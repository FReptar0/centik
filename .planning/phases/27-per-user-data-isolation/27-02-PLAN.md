For ALL 6 action files, apply these changes:

**Import change (all files):**
- Replace `import { getDefaultUserId } from '@/lib/auth-utils'` with `import { requireAuth } from '@/lib/auth-utils'`

**requireAuth() placement (CRITICAL -- all files):**
- Move auth call BEFORE the try/catch block: `const { userId } = await requireAuth()`
- redirect() inside try/catch is swallowed -- this is a security-critical placement requirement
- For functions that currently call getDefaultUserId() inside try/catch (createTransaction, createDebt, closePeriod, createIncomeSource, createCategory, upsertBudgets): extract `const { userId } = await requireAuth()` to BEFORE the try block
- For getTransactionFormData and getClosePeriodPreviewAction which have no try/catch: replace getDefaultUserId() with `const { userId } = await requireAuth()`

**IDOR fixes (specific files) -- replace findUnique with findFirst including userId:**

movimientos/actions.ts:
- `updateTransaction`: Replace `prisma.transaction.findUnique({ where: { id } })` with `prisma.transaction.findFirst({ where: { id, userId }, include: { period: true } })`. Keep the existing `update({ where: { id } })` since ownership is verified by findFirst.
- `deleteTransaction`: Replace `prisma.transaction.findUnique({ where: { id } })` with `prisma.transaction.findFirst({ where: { id, userId }, include: { period: true } })`. Keep `delete({ where: { id } })` since ownership is verified.

deudas/actions.ts:
- `updateDebt`: Add ownership check before update: `const existing = await prisma.debt.findFirst({ where: { id, userId } }); if (!existing) return { error: { _form: ['Deuda no encontrada'] } }`. Keep `update({ where: { id } })` after ownership verified.
- `updateDebtBalance`: Same pattern -- findFirst with userId, then update.
- `deleteDebt`: Same pattern -- findFirst with userId, then delete. Replace current P2025 catch with pre-check.

ingresos/actions.ts:
- `updateIncomeSource`: Add ownership check: `const existing = await prisma.incomeSource.findFirst({ where: { id, userId } }); if (!existing) return { error: { _form: ['Fuente de ingreso no encontrada'] } }`. Keep update.
- `deleteIncomeSource`: Same pattern -- findFirst, then delete. Replace P2025 catch with pre-check.

configuracion/actions.ts:
- `deleteCategory`: Replace `prisma.category.findUnique({ where: { id } })` with `prisma.category.findFirst({ where: { id, userId } })`.

historial/actions.ts:
- `closePeriod`: Replace `prisma.period.findUnique({ where: { id: periodId } })` with `prisma.period.findFirst({ where: { id: periodId, userId } })`.
- `reopenPeriod`: Same -- replace findUnique with `prisma.period.findFirst({ where: { id: periodId, userId } })`.
- `reopenPeriod` also needs requireAuth() added (currently has no auth call at all).

presupuesto/actions.ts:
- No IDOR fix needed -- upsertBudgets already uses findFirst with userId.
