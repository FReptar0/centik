    Create `src/app/configuracion/actions.ts` with `'use server'` directive.

    Replicate the Phase 5 Server Action pattern exactly:
    - Define `ActionResult` type (same union as Phase 5)
    - Define `getPrismaErrorCode` helper (same duck-typing pattern)
    - Define `revalidateCategoryPaths()` helper that calls `revalidatePath('/configuracion')` and `revalidatePath('/movimientos')` (transaction form uses categories)

    Implement two Server Actions:

    1. `createCategory(data: unknown): Promise<ActionResult>`
       - Parse with `createCategorySchema.safeParse(data)`
       - On validation failure: return `{ error: parsed.error.flatten().fieldErrors }`
       - Get the max sortOrder from existing categories and set new one to max + 1
       - Create with `prisma.category.create({ data: { name, icon, color, type, isDefault: false, sortOrder: maxSort + 1 } })`
       - Catch P2002: return `{ error: { name: ['Ya existe una categoria con ese nombre'] } }`
       - Call revalidateCategoryPaths()
       - Return `{ success: true }`

    2. `deleteCategory(id: string): Promise<ActionResult>`
       - First fetch the category to check isDefault: `prisma.category.findUnique({ where: { id }, select: { isDefault: true } })`
       - If not found: return `{ error: { _form: ['Categoria no encontrada'] } }`
       - If isDefault is true: return `{ error: { _form: ['No se pueden eliminar categorias predeterminadas'] } }`
       - Soft delete: `prisma.category.update({ where: { id }, data: { isActive: false } })` (soft delete preserves transaction history)
       - Call revalidateCategoryPaths()
       - Return `{ success: true }`

    IMPORTANT: Use soft delete (isActive=false) not hard delete, because categories have transactions referencing them. The UI filters by isActive=true.
