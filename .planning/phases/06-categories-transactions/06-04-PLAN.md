    **2. Update page.tsx** (Server Component)

    Replace the placeholder page. This is an async Server Component:

    ```typescript
    interface PageProps {
      searchParams: Promise<{
        month?: string; year?: string;
        type?: string; categoryId?: string;
        startDate?: string; endDate?: string;
        paymentMethod?: string; limit?: string;
      }>
    }
    ```

    Steps:
    1. Await searchParams (Next.js 16 async pattern)
    2. Resolve period: if month/year in params, fetch that period; otherwise getCurrentPeriod()
    3. Build Prisma where clause from filter params:
       ```typescript
       const where: Prisma.TransactionWhereInput = {
         periodId: period.id,
         ...(type && { type: type as TransactionType }),
         ...(categoryId && { categoryId }),
         ...(paymentMethod && { paymentMethod: paymentMethod as PaymentMethod }),
         ...(startDate || endDate ? {
           date: {
             ...(startDate && { gte: new Date(startDate) }),
             ...(endDate && { lte: new Date(endDate) }),
           }
         } : {}),
       }
       ```
    4. Fetch transactions with limit (default 25):
       ```typescript
       const limit = Math.min(Number(params.limit) || 25, 200)
       const [transactions, totalCount] = await Promise.all([
         prisma.transaction.findMany({
           where,
           include: { category: { select: { name: true, icon: true, color: true } } },
           orderBy: [{ date: 'desc' }, { createdAt: 'desc' }],
           take: limit,
         }),
         prisma.transaction.count({ where }),
       ])
       ```
    5. Fetch categories (for filters and form): `prisma.category.findMany({ where: { isActive: true }, orderBy: { sortOrder: 'asc' } })`
    6. Fetch income sources (for form): `prisma.incomeSource.findMany({ where: { isActive: true } })`
    7. Serialize BigInt fields: `serializeBigInts(transactions)` and `serializeBigInts(incomeSources)` (categories have no BigInt)
    8. Cast via unknown to typed arrays (same pattern as Phase 5 serializeBigInts cast)
    9. Return MovimientosClientWrapper with all data props

    **4. Update FAB.tsx**

    The FAB currently renders a placeholder modal. Update it to render the real TransactionForm.

    Challenge: FAB is rendered in the root layout (Server Component), but TransactionForm needs categories and incomeSources data. Two approaches:

    **Approach A (preferred -- simpler):** FAB stays as a client component that opens a modal. The modal content is the TransactionForm. FAB fetches categories/incomeSources data from a lightweight Server Action when it opens (lazy load on click). This avoids passing data through the root layout.

    Create a new Server Action in `src/app/movimientos/actions.ts`:
    ```typescript
    export async function getTransactionFormData(): Promise<{
      categories: Category[]
      incomeSources: SerializedIncomeSource[]
    }>
    ```
    This fetches active categories and income sources, serializes BigInts, and returns. FAB calls this on first open, caches in state.

    **Approach B:** Pass categories/incomeSources from root layout to FAB as props. This requires the root layout to fetch data and pass to a client boundary, which is more complex.

    Use Approach A. Update FAB.tsx:
    - Remove the placeholder modal content
    - On first open: call `getTransactionFormData()` to load categories and income sources
    - Store in state (loaded once, reused on subsequent opens -- revalidation handles freshness)
    - Render TransactionForm inside the existing Modal with the loaded data
    - While loading data on first open, show a brief loading spinner
    - The FAB's Modal wrapping is already there -- just replace the placeholder children with TransactionForm (but TransactionForm expects its own Modal via isOpen/onClose. Solution: remove Modal from FAB, let TransactionForm handle its own Modal. FAB just manages isOpen state and passes it to TransactionForm.)

    Revised FAB structure:
    ```typescript
    export default function FAB() {
      const [isOpen, setIsOpen] = useState(false)
      const [formData, setFormData] = useState<{ categories: Category[]; incomeSources: SerializedIncomeSource[] } | null>(null)

      async function handleOpen() {
        setIsOpen(true)
        if (!formData) {
          const data = await getTransactionFormData()
          setFormData(data)
        }
      }

      return (
        <>
          <button onClick={handleOpen} ...FAB styles...>
            <Plus />
          </button>
          {formData && (
            <TransactionForm
              isOpen={isOpen}
              onClose={() => setIsOpen(false)}
              categories={formData.categories}
              incomeSources={formData.incomeSources}
            />
          )}
        </>
      )
    }
    ```

    Add `getTransactionFormData` to src/app/movimientos/actions.ts (Server Action that fetches categories + incomeSources, serializes, returns).
