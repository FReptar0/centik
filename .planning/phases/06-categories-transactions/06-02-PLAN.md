    Create `src/app/movimientos/actions.ts` with `'use server'` directive.

    Replicate the Phase 5 Server Action pattern:
    - Define `ActionResult` type (same union)

    - Define `revalidateTransactionPaths()` helper: revalidatePath('/'), revalidatePath('/movimientos'), revalidatePath('/presupuesto')

    Implement three Server Actions:

    1. `createTransaction(data: unknown): Promise<ActionResult>`
       - Parse with `createTransactionSchema.safeParse(data)`
       - On validation failure: return field errors
       - Resolve period: call `getPeriodForDate(parsed.data.date)` from `@/lib/period`
       - **Closed-period check:** if `period.isClosed === true`, return `{ error: { _form: ['El periodo de este mes esta cerrado. Reabre el periodo para agregar transacciones.'] } }`
       - Create transaction:
         ```typescript
         prisma.transaction.create({
           data: {
             type: parsed.data.type,
             amount: BigInt(parsed.data.amount),
             categoryId: parsed.data.categoryId,
             date: new Date(parsed.data.date),
             description: parsed.data.description ?? null,
             paymentMethod: parsed.data.paymentMethod ?? null,
             notes: parsed.data.notes ?? null,
             incomeSourceId: parsed.data.incomeSourceId ?? null,
             periodId: period.id,
           },
         })
         ```
       - Call revalidateTransactionPaths()
       - Return `{ success: true }`

    2. `updateTransaction(id: string, data: unknown): Promise<ActionResult>`
       - Parse with `createTransactionSchema.safeParse(data)` (same schema for update)
       - On validation failure: return field errors
       - Fetch existing transaction with its period: `prisma.transaction.findUnique({ where: { id }, include: { period: true } })`
       - If not found: return `{ error: { _form: ['Transaccion no encontrada'] } }`
       - **Closed-period check:** if `existing.period.isClosed === true`, return closed-period error
       - Resolve new period for the updated date (in case date changed): `getPeriodForDate(parsed.data.date)`
       - If new period is closed, also reject
       - Update the transaction (including potentially new periodId if date changed months)
       - Call revalidateTransactionPaths()
       - Return `{ success: true }`

    3. `deleteTransaction(id: string): Promise<ActionResult>`
       - Fetch transaction with period: `prisma.transaction.findUnique({ where: { id }, include: { period: true } })`
       - If not found: return `{ error: { _form: ['Transaccion no encontrada'] } }`
       - **Closed-period check:** if `period.isClosed === true`, return closed-period error
       - Delete: `prisma.transaction.delete({ where: { id } })`
       - Call revalidateTransactionPaths()
       - Return `{ success: true }`
