## Gold-standard analog: prisma/seed.ts L1-42 — seedAdminUser (dev seed — DO NOT copy its demo-data helpers)
```
import 'dotenv/config'
import { PrismaClient } from '../generated/prisma/client'
import { PrismaPg } from '@prisma/adapter-pg'
import bcrypt from 'bcryptjs'

const adapter = new PrismaPg({
  connectionString: process.env.DATABASE_URL!,
})
const prisma = new PrismaClient({ adapter })

async function seedAdminUser(): Promise<string> {
  const email = process.env.ADMIN_EMAIL || 'fmemije00@gmail.com'
  const password = process.env.ADMIN_PASSWORD || 'centik-dev-2026'
  const hashedPassword = await bcrypt.hash(password, 12)

  const user = await prisma.user.upsert({
    where: { email },
    update: { isAdmin: true },
    create: {
      email,
      hashedPassword,
      isApproved: true,
      totpEnabled: false,
      isAdmin: true,
    },
  })

  console.log(`Seeded admin user: ${email}`)
  return user.id
}
```

### What to copy from the analog
- Adapter + PrismaClient bootstrap lines (L1-9).

### What MUST differ per D-09 / D-10 / D-12
1. **NO default fallbacks.** The dev seed defaults to `'fmemije00@gmail.com'` + `'centik-dev-2026'` if env vars are unset. The prod seed MUST throw in both cases. Minimum password length: 12 chars.
2. **No password rotation on existing admin.** The dev seed's `update: { isAdmin: true }` is close but the prod seed's update block must include `isApproved: true` as well — and MUST NOT include `hashedPassword`. Add a branch: if the user already exists, call `prisma.user.update({ where: { email }, data: { isAdmin: true, isApproved: true } })` and log "already exists — flags verified (password NOT rotated)". If the user does not exist, run `prisma.user.create({ ...with hashedPassword... })`.
3. **Zero demo data.** No calls to `seedCategories`, `seedDebts`, `seedIncomeSources`, `seedTransactions`, `seedBudgets`, `seedValueUnits`, `seedMonthlySummary`, `seedUnitRates`, `seedPeriods`. No writes to `prisma.category`, `prisma.debt`, `prisma.transaction`, `prisma.incomeSource`, `prisma.period`, `prisma.budget`, `prisma.valueUnit`, `prisma.unitRate`, `prisma.monthlySummary`, `prisma.asset`, `prisma.backupCode`.
4. **Hash BEFORE transaction.** Phase 28 P03 decision — 12-cost bcrypt is ~300ms and MUST NOT pin a DB connection. Compute `hashedPassword` in a standalone line BEFORE any prisma call.

    ```
    // prisma/seed.prod.ts — production seed (admin-only, idempotent, no rotate-on-exists).
    // Manual one-shot post-deploy: `vercel env pull .env.production && npm run db:seed:prod`.
    // NEVER wire into the Vercel build command (D-11).
    // Writes ONLY to the User table (D-12): zero demo categories/debts/transactions/etc.
    import 'dotenv/config'
    import { PrismaClient } from '../generated/prisma/client'
    import { PrismaPg } from '@prisma/adapter-pg'
    import bcrypt from 'bcryptjs'

    // D-09 — fail fast if operator forgot to set either env var.
    const email = process.env.ADMIN_EMAIL
    const password = process.env.ADMIN_PASSWORD
    if (!email) {
      throw new Error('ADMIN_EMAIL is required for production seed.')
    }
    if (!password || password.length < 12) {
      throw new Error('ADMIN_PASSWORD is required and must be at least 12 characters.')
    }

    // Uses DATABASE_URL (pooled). A single-row upsert is pooler-safe.
    // If the operator prefers the direct connection, swap to DIRECT_URL. Document both.
    const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL! })
    const prisma = new PrismaClient({ adapter })

    async function main(): Promise<void> {
      // D-10 — branch on existence: existing admin is a flags-only update (NO password rotation).
      const existing = await prisma.user.findUnique({
        where: { email: email! },
        select: { id: true },
      })

      if (existing) {
        await prisma.user.update({
          where: { email: email! },
          data: { isAdmin: true, isApproved: true },
        })
        console.log(
          `Admin user ${email} already exists -- flags verified (password NOT rotated).`,
        )
        return
      }

      // New admin: hash password BEFORE any $transaction to avoid connection pinning
      // (~300ms cost-12 bcrypt; Phase 28 P03 discipline).
      const hashedPassword = await bcrypt.hash(password!, 12)

      await prisma.user.create({
        data: {
          email: email!,
          hashedPassword,
          isAdmin: true,
          isApproved: true,
          totpEnabled: false,
        },
      })

      console.log(`Admin user ${email} created.`)
    }

    main()
      .then(async () => {
        await prisma.$disconnect()
      })
      .catch(async (err) => {
        console.error(err)
        await prisma.$disconnect()
        process.exit(1)
      })
    ```
