Add a `validateField` helper function that validates a single field using the corresponding Zod schema's partial parse (or manual per-field validation). For simplicity, run the full schema safeParse and extract only the error for the specific field:
```ts
function validateField(fieldName: string, formData: Record<string, unknown>) {
  const result = schema.safeParse(formData)
  if (result.success) {
    setErrors((prev) => {
      const next = { ...prev }
      delete next[fieldName]
      return next
    })
  } else {
    const fieldErrors = result.error.issues
      .filter((issue) => issue.path[0] === fieldName)
      .map((issue) => issue.message)
    setErrors((prev) => ({
      ...prev,
      [fieldName]: fieldErrors.length > 0 ? fieldErrors : prev[fieldName] ?? [],
    }))
  }
}
```

3. **Comma formatting on blur:** On blur, format the display value with commas for readability. On focus, strip commas back to raw number for editing.
```ts
function formatAmountDisplay(value: string): string {
  const num = parseFloat(value)
  if (isNaN(num)) return value
  return num.toLocaleString('es-MX', { minimumFractionDigits: 0, maximumFractionDigits: 2 })
}

function stripCommas(value: string): string {
  return value.replace(/,/g, '')
}
```

Use a `displayValue` state for each amount field that holds the formatted version, while the actual form state holds the raw numeric string. On blur: set displayValue to formatted. On focus: set displayValue to raw value.

4. **Ensure `inputMode="decimal"`** is present (most already have it).

const cleaned = value.replace(/[^0-9.]/g, '').replace(/(\..*)\./g, '$1')
