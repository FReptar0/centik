**1. Global focus-visible ring styles in globals.css:**

Add a global CSS rule for visible focus rings on all interactive elements. This ensures EVERY button, link, and input gets a consistent focus ring without needing per-component classes:

```css
/* Accessibility: visible focus rings (UX_RULES.md 9.1) */
:focus-visible {
  outline: 2px solid var(--color-accent, #22d3ee);
  outline-offset: 2px;
  border-radius: inherit;
}

/* Remove default focus outline for mouse users */
:focus:not(:focus-visible) {
  outline: none;
}
```

Check the Tailwind v4 variable naming. The project uses `@theme` with custom tokens. The accent color may be referenced as `--color-accent` or the raw hex `#22d3ee`. Inspect `src/app/globals.css` for the existing @theme block and use the correct variable name. If no CSS variable is defined for accent, use the raw hex value.

**2. Table semantic fixes:**

In `src/components/history/AnnualPivotTable.tsx`:
- Add `scope="col"` to all `<th>` elements in `<thead>`
- Add `scope="row"` to the first `<th>` or `<td>` in each `<tbody>` row if rows have a label column

In `src/components/budgets/BudgetTable.tsx`:
- Same: add `scope="col"` to all `<th>` elements in `<thead>`

**3. Semantic sections with aria-labelledby on pages:**

For each page that has distinct sections, wrap section content in `<section>` with `aria-labelledby` pointing to a heading `id`. This is a lightweight touch -- only add where natural section boundaries exist:

- **Dashboard (src/app/page.tsx):** The page likely has KPI grid, charts section, and recent transactions. Add `<section aria-labelledby="kpi-heading">`, `<section aria-labelledby="charts-heading">`, `<section aria-labelledby="recent-heading">` with corresponding `id` attributes on the heading elements (h2/h3). If no headings exist, add visually hidden `<h2>` with `sr-only` class.

- **Other pages:** Most pages are wrapped in ClientWrapper components making server-side section wrapping impractical. For these, add `aria-labelledby` only where it naturally fits without restructuring. At minimum, ensure each page's content is wrapped by `<main>` (already done in layout.tsx). Do NOT force-add sections if the page structure doesn't support it.

**4. Audit decorative vs functional icons:**

Grep for DynamicIcon and Lucide icon usage without aria-hidden or aria-label. Add:
- `aria-hidden="true"` to all decorative icons (category icons in lists, nav icons -- most already have this)
- `aria-label="descriptive text"` to functional icons (action buttons that are icon-only, like delete buttons)

Focus on icon-only buttons: if a button has ONLY an icon and no visible text, it MUST have `aria-label`. Check components: IncomeSourceCard edit/delete buttons, TransactionRow delete button, DebtCard expand toggle. Most of these have visible text alongside, but verify.
