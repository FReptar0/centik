    **2. Create src/lib/dashboard.ts with 5 query functions:**

    Each function uses Prisma aggregate/groupBy — NO loading of individual records for KPIs.

    `getDashboardKPIs(periodId: string)`: Use Promise.all for 4 parallel queries:
    - Monthly estimated income: `prisma.incomeSource.findMany({ where: { isActive: true }, select: { defaultAmount: true, frequency: true } })` then compute monthly equivalent using the same CASE logic as DATA_FLOW.md SQL (QUINCENAL x2, SEMANAL x4, MENSUAL x1, VARIABLE x1). Use BigInt arithmetic, not the income.ts helper (that takes serialized strings; here we have raw BigInt from Prisma).
    - Month expenses: `prisma.transaction.aggregate({ _sum: { amount: true }, where: { periodId, type: 'EXPENSE' } })`
    - Month income (real): `prisma.transaction.aggregate({ _sum: { amount: true }, where: { periodId, type: 'INCOME' } })`
    - Total debt: `prisma.debt.aggregate({ _sum: { currentBalance: true }, where: { isActive: true } })`
    - Compute derived values: available = monthlyIncome - monthExpenses, savingsRate, debtToIncomeRatio
    - Return DashboardKPIs with all BigInt fields serialized to strings via .toString()
    - Handle zero income: savingsRate and debtToIncomeRatio = 0 when income is 0

    `getCategoryExpenses(periodId: string)`: Use `prisma.transaction.groupBy({ by: ['categoryId'], _sum: { amount: true }, where: { periodId, type: 'EXPENSE' } })` then join category data with a separate findMany query (Prisma groupBy does not support include). Return CategoryExpense[] sorted by total descending.

    `getBudgetVsSpent(periodId: string)`: Query budgets for the period with category include, then LEFT JOIN spent amounts. Two approaches:
    - Approach A: findMany budgets with category include, then separate groupBy for expenses, merge in JS
    - Use approach A since Prisma cannot do the SQL LEFT JOIN in one query
    - Budget monthly = quincenalAmount * BigInt(2). Serialize to strings.

    `getMonthlyTrend()`: Query MonthlySummary with period include, order by period year/month, take last 6. Serialize BigInt fields to strings. For the current month (if not closed yet), compute live totals from transactions as a synthetic entry (so the trend chart shows current month progress).

    `getRecentTransactions(periodId: string)`: `prisma.transaction.findMany({ where: { periodId }, include: { category: { select: { name, icon, color } } }, orderBy: [{ date: 'desc' }, { createdAt: 'desc' }], take: 8 })`. Serialize BigInt amount to string.
