**### 4. Iconos de Categoria (Pixel-Art)**

Write the category icon aesthetic guidance:

**Philosophy:** 8x8 pixel-art inspired — simple, geometric, instantly recognizable at small sizes. The icons should feel like they belong on a Nothing Phone glyph interface, not a generic app.

**Library:** Lucide React (no custom icon set). The pixel-art aesthetic is achieved through icon selection and rendering parameters, not a custom font.

**Icon selection guidance:**
- Prefer geometric, angular icons over organic/curved ones
- Prefer thin-stroke variants over filled variants
- Prefer icons with straight lines and right angles when available
- Examples of "on-brand" Lucide icons: `Utensils` (geometric), `Zap` (angular), `Briefcase` (rectangular), `Car` (simple silhouette)
- Examples to avoid: Icons with excessive curves, ornate detail, or organic shapes

**Rendering rules:**
- `stroke-width: 1.5` (thinner than Lucide default of 2, more precise feel)
- `shape-rendering: crispEdges` on the SVG element for crisp pixel-aligned rendering without anti-aliasing smoothing
- Icon color: category color (solid) from the desaturated palette

**Container (already specced in Iconografia):**
- Background: category color at 12% opacity
- Size: 36x36px (lists), 40x40px (cards)
- Border-radius: 12px
- Cross-reference: "Especificacion de contenedor en Iconografia > Contenedores de Icono"

**### 5. Status Dot (Indicador de Estado)**

Write the status dot animation spec:

**Purpose:** A small LED-like indicator that communicates "live" or "active" state without text. Inspired by Nothing Phone's glyph LED indicators.

**Placement strategy** (Claude's discretion):
- Current period indicator in the period selector — a dot next to the active period signals "this is live"
- Active nav item in bottom tab bar — a dot below the active icon replaces text labels (pairs with UX-01)
- KPI cards showing real-time or recently-updated data — a dot near the value signals freshness

Use sparingly. Maximum 2-3 visible dots on any single screen. Overuse dilutes the signal.

**Visual spec:**
- Size: 4px solid circle
- Color: `--color-accent` (#CCFF00, chartreuse)
- No glow, no shadow, no ring — just the dot itself (consistent with shadow-free elevation philosophy)

**Animation:** Subtle continuous pulse. The dot breathes slowly — never frantic, never attention-grabbing. Communicates "alive" not "urgent."

**CSS implementation (ready to copy):**
```css
@keyframes status-pulse {
  0%, 100% {
    opacity: 1;
    transform: scale(1);
  }
  50% {
    opacity: 0.5;
    transform: scale(0.85);
  }
}

.status-dot {
  width: 4px;
  height: 4px;
  border-radius: 50%;
  background-color: var(--color-accent);
  animation: status-pulse 2.5s ease-in-out infinite;
}
```

**Duration:** 2.5s cycle — slow enough to feel calm, fast enough to be noticed.

**### 6. Micro-Animacion Pixel-Dissolve**

Write the pixel-dissolve animation spec:

**Purpose:** A scanline reveal effect that makes data appear as if being rendered by a dot-matrix printer or CRT scan. Used when new data appears on screen — KPIs updating, charts re-rendering, fresh transaction data loading.

**Visual description:** The element renders top-to-bottom in horizontal scanlines. The content appears in thin horizontal strips from top to bottom, as if being printed line by line. The effect should feel mechanical, precise, and retro — like a printer head moving across paper.

**Trigger contexts:**
- KPI card values updating after data refresh
- Chart re-rendering after period change
- New transaction appearing in a list
- Initial page data load (first paint of data-heavy components)

NOT for: navigation transitions, modal open/close, button interactions, hover states.

**Timing:**
- Duration: 400-600ms (default 500ms)
- Timing function: `steps(12, end)` — stepped progression for mechanical feel, NOT smooth easing
- 12 steps = 12 scanlines across the element height

**CSS implementation (ready to copy):**

Approach: CSS `clip-path` with stepped animation to reveal content in horizontal slices.

```css
@keyframes scanline-reveal {
  0% {
    clip-path: inset(0 0 100% 0);
  }
  100% {
    clip-path: inset(0 0 0% 0);
  }
}

.pixel-dissolve {
  animation: scanline-reveal 500ms steps(12, end) forwards;
}

/* Variant for data refresh (element already visible, re-triggers) */
.pixel-dissolve-refresh {
  animation: none;
}
.pixel-dissolve-refresh.updating {
  animation: scanline-reveal 500ms steps(12, end) forwards;
}
```

**Composability:** The animation applies to the entire element. For KPI cards, apply to the value container. For charts, apply to the chart wrapper. For list items, apply to individual row elements.

**Reduced motion:** Respect `prefers-reduced-motion`. When reduced motion is preferred, skip the animation entirely — content appears immediately.

```css
@media (prefers-reduced-motion: reduce) {
  .pixel-dissolve,
  .pixel-dissolve-refresh.updating {
    animation: none;
    clip-path: none;
  }
}
```
