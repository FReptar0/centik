**Progress Bars spec (replaces existing):**

Rename section to: "### Progress Bars (Battery-Bar)"

Description: Replaces ALL smooth/continuous progress bars throughout the app. Every percentage indicator (budget progress, credit utilization, debt payoff) uses this segmented battery-bar style.

Structure:
- Container: Background at semantic color 12% opacity. Height: 6px (compact variant) or 8px (detailed variant). Full width of parent.
- Segments: Always exactly 10 rectangular segments. Each segment = 10% of the total. 2px gaps between segments. Segments are flat rectangles (no border-radius).
- Fill direction: Left to right.

Traffic-light color system (applied per-segment):
- `--color-accent` (#CCFF00, chartreuse) for segments representing < 80% fill
- `--color-warning` (#FF9100, orange) for segments representing 80-99% fill
- `--color-negative` (#FF3333, red) for segments representing 100%+ fill

Overflow behavior (100%+): All 10 segments fill with `--color-negative` (red). Display overage as text to the right of the bar: "+15%" in `--color-negative` at Meta level (11px). This makes it immediately clear the budget/limit is exceeded without needing a separate indicator.

Examples:
- 45% budget spent: 4 chartreuse segments filled, 1 half-filled chartreuse, 5 empty
- 85% budget spent: 8 chartreuse segments, then the 9th segment in orange (crossing 80% threshold), 1 empty
- 110% over budget: All 10 segments in red + "+10%" text

Note: Half-filled segments show partial width within the segment space. The gap between segments is always maintained.

Container CSS concept:
```
display: flex; gap: 2px; align-items: center;
background: {semantic-color at 12% opacity};
padding: 2px;
```

Individual segment CSS concept:
```
flex: 1; height: 100%;
background: {color based on position in traffic-light system};
border-radius: 0; /* flat rectangles */
transition: background-color 300ms ease;
```
