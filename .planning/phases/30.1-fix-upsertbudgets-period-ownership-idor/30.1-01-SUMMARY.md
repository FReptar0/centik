# Plan 30.1-01 — Fix upsertBudgets period-ownership IDOR

## Outcome

Phase 30.1 is complete. The partial-IDOR in `upsertBudgetsAction` surfaced by Plan 30-05 is closed via the canonical Phase 27 `findFirst({ where: { id, userId } })` pattern applied to both the `periodId` argument AND every submitted `categoryId` in `entries[]`. Integration tests prove no stale cross-user Budget row can be created under either attack vector. Unit tests prove both rejection paths return the ambiguous `{ error: { _form: [...] } }` shape.

## Deliverables

### Task 1 — Production guards + unit tests

**`src/app/(app)/presupuesto/actions.ts`** (modified):
- Period ownership guard BEFORE the `try {}` block:
  ```typescript
  const period = await prisma.period.findFirst({
    where: { id: periodId, userId },
    select: { id: true },
  })
  if (!period) {
    return { error: { _form: ['Periodo no encontrado'] } }
  }
  ```
- Batched category ownership guard (single round-trip for all submitted ids):
  ```typescript
  const submittedCategoryIds = parsed.data.entries.map((e) => e.categoryId)
  const ownedCategories = await prisma.category.findMany({
    where: { id: { in: submittedCategoryIds }, userId },
    select: { id: true },
  })
  if (ownedCategories.length !== submittedCategoryIds.length) {
    return { error: { _form: ['Categoria no encontrada'] } }
  }
  ```
- Existing `Promise.all(parsed.data.entries.map(...))` upsert logic preserved verbatim.
- Error strings are ambiguous Spanish — no identity leak about which specific id failed (Phase 26/28/29 oracle-resistance rule).

**`src/app/(app)/presupuesto/actions.test.ts`** (extended):
- New module-level `vi.fn()` handles `mockPeriodFindFirst` + `mockCategoryFindMany`; extended the `vi.mock('@/lib/prisma', ...)` block with `period.findFirst` + `category.findMany` entries.
- New `beforeEach` default seeds both guards as passing so the existing 6 happy-path + validation tests keep green.
- 2 new `it()` tests cover the rejection paths:
  - `rejects with _form error when periodId does not belong to the authenticated user`
  - `rejects with _form error when any submitted categoryId is not owned by the authenticated user`
