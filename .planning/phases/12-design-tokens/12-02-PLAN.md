**Typography section ("## Tipografia"):**

1. **Font Stack** — per locked decisions:
   - Monospaced: IBM Plex Mono for ALL financial numbers everywhere (tables, lists, cards, inputs, KPIs, not just display amounts)
   - Geometric sans: Satoshi for headings, body text, labels
   - Document the CSS variable names:
     - --font-sans: 'Satoshi', system-ui, -apple-system, sans-serif
     - --font-mono: 'IBM Plex Mono', 'Fira Code', monospace
   - Note: Satoshi loaded via next/font (replaces DM Sans). IBM Plex Mono loaded via Google Fonts or next/font (replaces JetBrains Mono).

2. **5-Level Type Hierarchy** — per locked decision (compromise between Glyph Finance's 3-level and old 8-size scale):
   - Display: 36px/40px, Satoshi Bold (700) — KPI hero numbers, dashboard totals. Used sparingly (1-2 per page max)
   - Heading: 20px/28px, Satoshi Semibold (600) — page titles, section headers, card titles
   - Body: 14px/20px, Satoshi Regular (400) or Medium (500) — primary text, descriptions, table content
   - Label: 12px/16px, Satoshi Medium (500), uppercase, letter-spacing +2px, --color-text-secondary — form labels, column headers, metadata categories
   - Meta: 11px/14px, Satoshi Regular (400), --color-text-tertiary — timestamps, IDs, auxiliary info

3. **Financial Number Rules:**
   - ALL financial numbers use IBM Plex Mono at weight 600 or 700
   - Always use tabular-nums for column alignment
   - The dollar sign "$" is displayed at a smaller font-size and --color-text-tertiary (muted) — e.g., in a 36px Display amount, the "$" might be 24px and muted
   - Positive amounts: --color-positive
   - Negative amounts: --color-negative
   - Neutral amounts: --color-text-primary (#E8E8E8)

4. **Metadata/Label Style** — per locked decision:
   - Uppercase text
   - Letter-spacing: +2px (tracking-widest equivalent)
   - Smaller size (Label or Meta level)
   - Muted color (--color-text-secondary or --color-text-tertiary)
   - Used for: form labels, table column headers, section subtitles, badge text

5. **Typography Rules:**
   - Labels/descriptions always have less visual weight than the value they describe
   - No ALL CAPS for body text — only for Label-level metadata (max 3 words)
   - Minimum line-height of 1.4 for readable text blocks

**Spacing and Radius section ("## Espaciado y Radios"):**

1. **Spacing Scale** — based on 4px base, with Glyph Finance adjustments per locked decisions:
   - 4px (1): minimum inline spacing
   - 8px (2): icon-to-text gap, badge internal padding
   - 12px (3): gap between elements in compact lists, grid gap between cards
   - 16px (4): page margins (mobile), section title to content gap
   - 20px (5): card internal padding (standard)
   - 24px (6): card internal padding (generous), section separation within cards
   - 32px (8): separation between cards or page sections

2. **Spacing Rules** — updated values per CONTEXT.md:
   - Cards: 20-24px internal padding (standard 20px, generous 24px for KPI cards)
   - Card grid gap: 12px (was 16px)
   - Page margins: 16px on mobile, 24px on desktop
   - Table cells: 12px vertical, 16px horizontal
   - Form field vertical gap: 14px
   - Modal padding: 24px (was 28px)

3. **Radius Scale** — per locked decisions:
   - --radius-sm: 8px (badges, tags, small elements)
   - --radius-md: 12px (buttons, inputs — was 8px)
   - --radius-lg: 16px (cards — was 12px)
   - --radius-xl: 24px (modals — was 16px)
   - --radius-full: 9999px (circular elements, pill badges)

Note: All radii have increased compared to the old system. Buttons are now 12px (pill-ish), cards 16px (more rounded), modals 24px (significantly more rounded).
