**BudgetTable.tsx** -- Editable budget configuration table
- Props: `budgets: BudgetWithSpent[]`, `onSave: (entries: { categoryId: string; quincenalAmount: string }[]) => Promise<void>`, `isClosed: boolean`
- Client component ('use client')
- Local state: `editValues: Record<string, string>` mapping categoryId -> peso display value (not centavos)
- Initialize editValues from budgets on mount using a helper that converts centavos to pesos: `(Number(quincenalAmount) / 100).toString()` or empty string if zero
- Table structure (desktop): sticky header with columns: Categoria, Quincenal, Mensual, Semestral, Anual
- Category column: DynamicIcon (16px) + category name, with category color on the icon container (same pattern as TransactionRow)
- Quincenal column: text input (right-aligned, "$" prefix visual, inputMode="decimal") -- ONLY this column is editable
- Mensual column: readonly display of formatMoney(toCents(editValue) then BigInt * 2 toString) -- calculate from current input
- Semestral column: formatMoney of quincenal * 2 * 6 = quincenal * 12
- Anual column: formatMoney of quincenal * 2 * 12 = quincenal * 24
- For calculated columns: if toCents would throw (invalid input), show "--"
- Save button at bottom: calls onSave with entries mapped from editValues (toCents for each)
- If isClosed, inputs are disabled and save button hidden
- Mobile adaptation: hide Semestral and Anual columns on mobile (hidden md:table-cell), keep Categoria + Quincenal + Mensual
- Table styling per STYLE_GUIDE 8.5: header bg-bg-primary, text-text-secondary uppercase text-xs tracking-wide, cells py-3 px-4, border-b border-border-default
- Save button: bg-accent text-bg-primary rounded-lg px-4 py-2 font-semibold, with "Guardando..." state using useTransition or local isPending state

**BudgetProgressList.tsx** -- Progress bars per category showing % spent
- Props: `budgets: BudgetWithSpent[]`
- Client component ('use client') -- needed for getBudgetColor import and dynamic styling
- For each budget row:
  - Category info: DynamicIcon (16px) in colored container + name
  - Right side: spent / budgeted amounts (formatMoney each), e.g. "$3,200.00 / $5,000.00"
  - Progress bar below: container 6px height, bg of the budget color at 12% opacity, filled bar at the budget color
  - Percentage text below the bar: same color as the bar (per UX_RULES 10.3: "El texto del porcentaje adopta el mismo color que la barra")
  - Calculate percentUsed: monthly budget = BigInt(quincenalAmount) * 2, spent = BigInt(spent). If monthly > 0: (Number(spent) / Number(monthly)) * 100, else 0
  - Use getBudgetColor(percentUsed) to get 'positive' | 'warning' | 'negative'
  - Map color names to Tailwind classes: positive -> text-positive / bg-positive, warning -> text-warning / bg-warning, negative -> text-negative / bg-negative
  - For the subtle background of the progress bar container, use the corresponding subtle variant: bg-positive-subtle, bg-warning-subtle, bg-negative-subtle
  - Progress bar width: `min(percentUsed, 200)%` capped visually at 100% width via overflow-hidden on container, but show actual percentage in text (e.g., "135%")
  - Progress bar transition: `transition-all duration-500 ease-out` (per STYLE_GUIDE 8.7)
- Empty state: if budgets is empty, show DynamicIcon "piggy-bank" (32px) + "Sin presupuesto configurado" + description text
- Layout: vertical stack with gap-4 between rows

**BudgetSummaryRow.tsx** -- Total row comparing income vs budget
- Props: `budgets: BudgetWithSpent[]`, `quincenalIncome: string` (centavos)
- Not a client component unless needed; can be server or client -- make it client for consistency since parent is client wrapper
- Calculate total quincenal budget: sum all BigInt(quincenalAmount) across budgets
- Calculate surplus/deficit: BigInt(quincenalIncome) - totalQuincenalBudget

  - Row 2: "Sobrante" or "Faltante" -> formatMoney(abs value), colored green (sobrante) or red (faltante)
  - Use text-positive for sobrante, text-negative for faltante
  - Calculate monthly (x2), show as smaller secondary info: "Mensual: $X sobrante/faltante"

- Fetch budgets: call getBudgetsWithSpent(period.id)

- Serialize budgets data (BudgetWithSpent already has string amounts from getBudgetsWithSpent, so no additional serialization needed)
