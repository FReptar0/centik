  **Step 2: Create `src/lib/validators.ts` with all 8 schemas.**

  At the top of the file:
  ```typescript
  import { z } from 'zod'
  import {
    TransactionType,
    Frequency,
    DebtType,
    PaymentMethod,
    CategoryType,
    AssetCategory,
  } from '@/types'

  // Set Spanish as fallback locale for unspecified error messages
  z.config(z.locales.es())

  // Shared regex: non-negative integer string (centavos / basis points)
  const amountRegex = /^\d+$/
  ```

  Then define all 8 schemas with JSDoc comments explaining which endpoint each validates. Use these Zod v4 patterns:

  - `z.enum(TransactionType, { error: '...' })` for all Prisma enums (NOT z.nativeEnum)
  - `z.string({ error: (iss) => iss.input === undefined ? 'Campo requerido' : 'Debe ser texto' })` for required strings where you need different messages for missing vs wrong type
  - `.regex(amountRegex, { error: '...' })` for amount validation
  - `z.iso.date({ error: '...' })` for date strings
  - `.refine()` with `{ error: '...', path: ['field'] }` for cross-field validation (e.g., creditLimit required when type is CREDIT_CARD)

  Schema specifications:

  **createTransactionSchema:** type (TransactionType enum), amount (string, regex digits), categoryId (string required), date (z.iso.date), description (string optional), paymentMethod (PaymentMethod enum optional), notes (string optional), incomeSourceId (string optional).

  **createDebtSchema:** name (string min 1), type (DebtType enum), currentBalance (string regex), creditLimit (string regex optional), annualRate (number int nonnegative), minimumPayment (string regex optional), monthlyPayment (string regex optional), originalAmount (string regex optional), remainingMonths (number int nonnegative optional), cutOffDay (number int 1-31 optional), paymentDueDay (number int 1-31 optional). Refine: if type is CREDIT_CARD, creditLimit must be defined.

  **updateDebtBalanceSchema:** currentBalance (string regex).

  **createBudgetSchema:** entries (array min 1 of objects with categoryId string and quincenalAmount string regex).

  **createIncomeSourceSchema:** name (string min 1), defaultAmount (string regex), frequency (Frequency enum), type (z.string min 1 -- it's a plain string, not an enum, per Phase 2 decision).

  **createCategorySchema:** name (string min 1), icon (string min 1), color (string regex /^#[0-9a-fA-F]{6}$/), type (CategoryType enum).

  **createAssetSchema:** name (string min 1), unitId (string), amount (string regex), category (AssetCategory enum), institution (string optional), notes (string optional).

  **createValueUnitSchema:** code (string min 1), name (string min 1), precision (number int nonnegative), providerUrl (string optional), providerPath (string optional), refreshInterval (number int nonnegative, default 0 via schema or optional).
