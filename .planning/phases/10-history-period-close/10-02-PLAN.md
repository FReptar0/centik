    'use client' component that manages:
    - Year selection state (default to current year or latest available year).
    - Close modal state (isOpen, selectedPeriodId, selectedMonth, selectedYear).
    - Preview data state (loaded via getClosePeriodPreview Server Action when modal opens).
    - Loading states for close and reopen operations.

    Props: `{ initialYear: number; availableYears: number[]; data: MonthSlot[]; periods: { id: string; month: number; year: number; isClosed: boolean }[]; currentPeriodId: string | null }`.

    Flow:
    1. Renders PageHeader with title "Historial" (no PeriodSelector -- historial has its own year selector). No closedBanner on historial page since it shows ALL months.
    2. Renders YearSelector.
    3. Renders AnnualPivotTable with data and periods.
    4. When user clicks "Cerrar" on a month in the table, open CloseConfirmationModal.
    5. On modal open, call getClosePeriodPreview(periodId) as a Server Action to load preview data.
    6. On confirm, call closePeriod(periodId). On success, use router.refresh() to reload data.
    7. On year change from YearSelector, update URL with ?year=XXXX using router.replace to trigger server re-render (same pattern as PeriodSelector).
    8. For reopen in the pivot table: when user clicks "Reabrir" on a closed month, call reopenPeriod(periodId) and router.refresh(). Reopen is reversible (user can close again), so direct action without extra confirmation is acceptable.

The reopen button also lives in PageHeader closedBanner on pages like /movimientos and /presupuesto when viewing a closed period (enabled by the new reopenAction prop). For the historial page, the pivot table itself has the "Reabrir" link next to closed month headers.
