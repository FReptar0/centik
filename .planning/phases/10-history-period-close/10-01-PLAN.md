  <action>
    Create src/app/historial/actions.ts as a 'use server' module following the established Server Action pattern (ActionResult type, Zod validation where needed, try/catch with generic error messages).

    **closePeriod(periodId: string): Promise&lt;ActionResult&gt;**

    1. Fetch the period by ID (include monthlySummary to check if already exists).
    2. If not found, return error.
    3. If period.isClosed, return error: "Este periodo ya esta cerrado".
    4. Execute prisma.$transaction(async (tx) => { ... }):
       a. Compute totals using tx.transaction.aggregate (same logic as getClosePeriodPreview — income, expenses).
       b. Compute debt totals using tx.debt.aggregate.
       c. Calculate totalSavings, savingsRate, debtPayments (BigInt(0) for MVP).
       d. tx.monthlySummary.create({ data: { periodId, totalIncome, totalExpenses, totalSavings, savingsRate, debtAtClose, debtPayments } })
       e. tx.period.update({ where: { id: periodId }, data: { isClosed: true, closedAt: new Date() } })
       f. Calculate next month/year (handle Dec->Jan year wrap).
       g. tx.period.upsert for next period — use month_year unique constraint. Upsert so it's safe if next period already exists.
       h. Get next period ID from upsert result.
       i. Copy budgets: tx.budget.findMany({ where: { periodId } }), then tx.budget.createMany with new periodId. Skip if no budgets or if next period already has budgets (check count first to be idempotent).
    5. revalidatePath for /, /historial, /presupuesto, /movimientos.
    6. Return { success: true }.

    **reopenPeriod(periodId: string): Promise&lt;ActionResult&gt;**

    1. Fetch period by ID.
    2. If not found, return error.
    3. If !period.isClosed, return error: "Este periodo no esta cerrado".
    4. Delete MonthlySummary where periodId.
    5. Update period: isClosed=false, closedAt=null.
    6. revalidatePath for /, /historial, /presupuesto, /movimientos.
    7. Return { success: true }.

    IMPORTANT: The $transaction in closePeriod must handle the budget copy within the transaction to maintain atomicity. Do NOT call copyBudgetsFromPreviousPeriod from budget.ts since it uses its own prisma client outside the transaction. Instead, inline the budget copy logic within the $transaction using the tx client.

    - For $transaction tests, mock it to call the callback with a mock tx object
    - Mock next/cache revalidatePath
    - Test all behaviors listed above
