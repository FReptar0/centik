    1. Install dependencies: `npm install next-auth@beta @auth/prisma-adapter --legacy-peer-deps` (try without --legacy-peer-deps first; only add if peer dep conflict)

    2. Generate AUTH_SECRET: `npx auth secret` will create/append to .env.local. Alternatively, generate with `openssl rand -base64 33` and add to .env as `AUTH_SECRET=<value>`. Also add `AUTH_SECRET=your-auth-secret-here` to .env.example.

    3. Create `src/auth.ts` -- Auth.js v5 central configuration:
       - Import NextAuth from 'next-auth', Credentials from 'next-auth/providers/credentials', PrismaAdapter from '@auth/prisma-adapter', bcrypt from 'bcryptjs', prisma from '@/lib/prisma'
       - IMPORTANT: Export authorize and callback functions as named exports for testability:
         - `export async function authorizeUser(credentials: { email?: string; password?: string } | undefined)` -- the core logic
         - `export async function jwtCallback({ token, user }: { token: JWT; user?: User | AdapterUser })` -- JWT callback
         - `export async function sessionCallback({ session, token }: { session: Session; token: JWT })` -- session callback
       - Export `{ handlers, signIn, signOut, auth }` from NextAuth({...})
       - adapter: PrismaAdapter(prisma)
       - session: { strategy: 'jwt', maxAge: 30 * 24 * 60 * 60 } (30 days per user decision)
       - pages: { signIn: '/login' }
       - providers: [Credentials({...})] with authorize: authorizeUser
       - authorizeUser logic:
         a. Returns null if credentials.email or credentials.password missing
         b. Queries prisma.user.findUnique by email, selecting id, email, name, hashedPassword, isApproved
         c. Returns null if user not found, or hashedPassword is null
         d. Returns null if !user.isApproved (block unapproved users)
         e. Uses bcrypt.compare(password, hashedPassword) -- returns null if invalid
         f. Returns { id: user.id, email: user.email, name: user.name } on success
       - callbacks: { jwt: jwtCallback, session: sessionCallback }
       - jwtCallback: if (user) { token.userId = user.id } -- only on initial sign-in; return token
       - sessionCallback: session.user.id = token.userId as string; return session
       - trustHost: true (for local dev)

    4. Create `src/proxy.ts` -- Route protection:
       - Import auth from '@/auth' and NextResponse from 'next/server'
       - Define publicPaths = ['/login', '/register']
       - Export proxy = auth((req) => { ... }) wrapping the logic:
         a. Extract pathname from req.nextUrl
         b. If pathname starts with any publicPaths entry:
            - If req.auth exists (already authenticated), redirect to '/'
            - Otherwise NextResponse.next()
         c. If !req.auth (unauthenticated on protected route):
            - Create loginUrl = new URL('/login', req.nextUrl.origin)
            - Set callbackUrl search param to pathname (per user decision: post-login redirect to callbackUrl)
            - Return NextResponse.redirect(loginUrl)
         d. Otherwise NextResponse.next()
       - Export config = { matcher: ['/((?!api/auth|_next/static|_next/image|favicon.ico|fonts).*)'] }

    5. Create `src/types/next-auth.d.ts` -- TypeScript module augmentation:
       - Augment 'next-auth' module: Session interface with user: { id: string } & DefaultSession['user']
       - Augment 'next-auth/jwt' module: JWT interface with userId?: string

    6. Create `src/app/api/auth/[...nextauth]/route.ts`:
       - `import { handlers } from '@/auth'`
       - `export const { GET, POST } = handlers`

    7. Add loginSchema to `src/lib/validators.ts`:
       - Read the existing file first
       - Add at the end: `/** Validates login form submission */`
       - `export const loginSchema = z.object({ email: z.string({ error: 'El correo es requerido' }).email({ error: 'Correo electronico no valido' }).trim(), password: z.string({ error: 'La contrasena es requerida' }).min(1, { error: 'La contrasena es requerida' }) })`

    8. Extend FloatingInput type prop union:
       - Read `src/components/ui/FloatingInput.tsx` first
       - Change the type prop from `'text' | 'number' | 'decimal' | 'date'` to `'text' | 'number' | 'decimal' | 'date' | 'password' | 'email'`
       - No other changes needed -- the component's behavior is input-type agnostic (per research open question #1)

    IMPORTANT: Do NOT add `AUTH_URL` or `AUTH_TRUST_HOST` env vars -- Auth.js v5 handles these automatically in development.
