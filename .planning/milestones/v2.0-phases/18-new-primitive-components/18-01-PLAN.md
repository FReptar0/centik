**Props interface:**
```typescript
interface BatteryBarProps {
  value: number           // 0-100+ percentage (already calculated by caller)
  max?: number            // Optional max display value (default 100), used only for ARIA
  variant?: 'compact' | 'detailed'  // 6px vs 8px height (default: 'compact')
  thresholds?: { warning: number; danger: number }  // default: { warning: 80, danger: 100 }
  className?: string
}
```

**Implementation details:**
- "use client" directive (no interactivity, but may be used inside client components)
- Container: flex row with 2px gap, full width
- 10 segments: each is a `<div>` with flex-1, flat rectangle (no border-radius), height from variant
- Segment fill logic: `filledCount = Math.min(Math.floor(value / 10), 10)`, partial fill for fractional segment via width percentage
- Color per segment position: if segment index < (warning threshold / 10), use accent; if < (danger threshold / 10), use warning; else use negative
- Overflow (value > danger threshold): all 10 segments negative color, append "+{excess}%" text in text-negative at 11px font size
- Empty segments: use bg-surface-elevated (or transparent with 12% opacity accent background per STYLE_GUIDE: "Fondo a color semantico al 12% de opacidad")
- ARIA: role="progressbar", aria-valuenow={value}, aria-valuemin={0}, aria-valuemax={max || 100}, aria-label provided via props or default "Progress"
- Each segment rendered in a containing div that has the background track color, with an inner div for the fill

**Segment color algorithm (per STYLE_GUIDE spec):**
The color is determined by the segment's cumulative position, not overall percentage:
- Segment represents range `[segIndex * 10, (segIndex + 1) * 10]`
- If the END of that range (i.e., `(segIndex + 1) * 10`) <= warning threshold: accent
- If the END of that range <= danger threshold: warning
- Else: negative
- For overflow (value > danger): ALL filled segments use negative color
