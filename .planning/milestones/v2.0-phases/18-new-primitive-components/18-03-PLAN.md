**Implementation details:**
- This is a purely visual/decorative component -- no "use client" needed (no state, no event handlers). However, since it uses animation classes that consumers may toggle, keep it as a simple function component without the directive. If the consumer is a client component, it will work fine.
- Render: `<div aria-hidden="true" className={cn("h-1 w-1 rounded-full bg-accent animate-status-pulse", className)} />`
- The `h-1 w-1` Tailwind classes produce 4px x 4px (Tailwind's spacing scale: 1 = 0.25rem = 4px at default root font size)
- The `animate-status-pulse` class references the `--animate-status-pulse` token defined in globals.css @theme block, which maps to `status-pulse 2.5s ease-in-out infinite`
- The prefers-reduced-motion override in globals.css handles disabling the animation -- no extra work needed in the component itself
- No glow, no shadow, no ring -- just the dot (per STYLE_GUIDE.md spec)

This component is intentionally minimal. Its value is semantic consistency -- every pulsing dot in the app uses the same component rather than ad-hoc div styling.

**Props interface:**
```typescript
interface TogglePillsProps {
  options: { value: string; label: string }[]
  value: string          // Currently selected value
  onChange: (value: string) => void
  className?: string     // Container className for layout positioning
}
```

**Implementation details:**
- "use client" directive (handles click events)
- Container: `<div className={cn("flex gap-1", className)} role="radiogroup">`
  - Using role="radiogroup" for accessibility since this is a single-select control
- Each option: `<button>` element with:
  - `type="button"` (never submit a form)
  - `role="radio"` and `aria-checked={isActive}` for accessibility
  - `onClick={() => onChange(option.value)}`
  - Active state classes: `bg-accent text-black font-semibold rounded-full`
  - Inactive state classes: `bg-transparent text-text-secondary rounded-full`
  - Shared classes: `px-4 py-2 text-sm rounded-full transition-all duration-200 active:scale-[0.98]`
  - Min height: `min-h-[40px]` per spec
- Use `cn()` to compose conditional classes based on `option.value === value`

**Press state:** `active:scale-[0.98]` for the tactile switch feel per STYLE_GUIDE.md button spec.

**Accessibility note:** Using role="radiogroup" on container and role="radio" + aria-checked on each button communicates that this is a single-selection group, which is semantically accurate for the Expense/Income toggle use case.
