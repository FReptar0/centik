  <action>
    Replace all box-style `<input>` elements with `<FloatingInput>` in every form across the app. Import FloatingInput from `@/components/ui/FloatingInput`. The key transformation pattern:

    **BEFORE (current pattern):**
    ```tsx
    <label htmlFor="x" className="block text-xs ...">Label</label>
    <input id="x" className="rounded-lg border bg-transparent px-3 py-2.5 ..." />
    {errors.x && <p className="text-xs text-negative">...</p>}
    ```

    **AFTER (FloatingInput pattern):**
    ```tsx
    <FloatingInput
      label="Label"
      value={value}
      onChange={setValue}
      error={errors.x?.[0]}
      prefix="$"  // only for money fields
      suffix="%"  // only for rate fields
      optional    // only for optional fields
    />
    ```

    **IncomeSourceForm.tsx:**
    - Replace "Nombre" input with `<FloatingInput label="Nombre" value={name} onChange={setName} error={errors.name?.[0]} />`
    - Replace "Monto por defecto" input with `<FloatingInput label="Monto por defecto" value={amount} onChange={...} prefix="$" error={errors.defaultAmount?.[0]} />`
    - For amount: since FloatingInput uses `onChange(value: string)` directly, remove the `cleanAmountInput` wrapper from the onChange and apply cleaning inside a custom handler. FloatingInput's onChange provides the raw string -- clean it before setting state.
    - The amount field currently uses displayAmount/amount dual state for comma formatting. With FloatingInput, simplify: keep amount as the raw value, handle formatting in a wrapper function that cleans then sets state. FloatingInput's value prop takes the raw amount string.
    - Frequency and Type pill selectors are NOT inputs -- they stay as button grids (they are already pill-shaped per Phase 19).
    - Remove the local FormField/AmountField helpers that were used for the old input pattern.
    - Remove the `<label>` elements and error `<p>` tags that FloatingInput now handles internally.

    **DebtForm.tsx:**
    - Replace "Nombre" input with FloatingInput.
    - Replace "Saldo actual" amount input with `<FloatingInput label="Saldo actual" value={currentBalance} onChange={...} prefix="$" error={errors.currentBalance?.[0]} />`
    - Replace "Tasa anual" input with `<FloatingInput label="Tasa anual" value={annualRate} onChange={...} suffix="%" error={errors.annualRate?.[0]} />`
    - Replace ALL credit card fields (creditLimit, minimumPayment, cutOffDay, paymentDueDay) with FloatingInput. Amount fields get prefix="$". Day fields are plain text with no prefix/suffix.
    - Replace ALL loan fields (originalAmount, monthlyPayment, remainingMonths) with FloatingInput.
    - Remove the local `FormField` and `AmountField` helper components entirely -- FloatingInput replaces their purpose.
    - Remove the shared `inputClass` constant.
    - For amount fields that had displayAmount/rawAmount dual-state for comma formatting: simplify to single value state with FloatingInput. The user types raw values (no comma formatting in this component). FloatingInput handles display naturally.

    **CategoryForm.tsx:**
    - Replace "Nombre" input with `<FloatingInput label="Nombre" value={name} onChange={...} error={errors.name?.[0]} />`
    - Icon selector and Color selector grids are NOT inputs -- they stay as button grids.
    - Remove the `<label>` and error `<p>` wrapping the name input.

    **BudgetTable.tsx:**
    - The quincenal amount inputs inside table cells need special treatment. Each row has a small inline amount input.
    - Replace the current `<input>` in each table row with `<FloatingInput label="" value={pesoValue} onChange={(v) => handleChange(budget.categoryId, cleanAmountInput(v))} prefix="$" disabled={isClosed} className="w-28" />`
    - Since the table context is compact, use label="" (empty label -- the column header already labels it). FloatingInput with empty label and a value will show the underline-only style without a floating label.
    - Remove the wrapping `<div className="relative">` and the `$` span that FloatingInput now handles via prefix prop.
    - IMPORTANT: BudgetTable uses `cleanAmountInput` in the onChange -- keep that cleaning logic, just call it inside the FloatingInput onChange callback.

    **TransactionFilters.tsx:**
    - The date range inputs (startDate, endDate) are the only actual `<input>` elements.
    - Replace with `<FloatingInput type="date" label="Desde" value={activeFilters.startDate ?? ''} onChange={(v) => setDateFilter('startDate', v)} />`
    - And `<FloatingInput type="date" label="Hasta" value={activeFilters.endDate ?? ''} onChange={(v) => setDateFilter('endDate', v)} />`
    - The filter chips and dropdown buttons are NOT inputs -- they stay as-is.
    - Remove the sr-only labels and surrounding div styling for the date inputs; FloatingInput handles its own layout.

    **For ALL forms -- validation integration:**
    - FloatingInput calls onChange with the string value directly. For fields that need validation-on-change (when touched), wrap the onChange: `onChange={(v) => { setValue(v); if (touched.has('field')) revalidate(v) }}`
    - FloatingInput does NOT have onBlur callback in its interface. For blur-based validation, the current approach of handleBlur needs adjusting. Two options:
      a) Add an onBlur handler to the component (but it is not in FloatingInput's props).
      b) Since FloatingInput already manages focus internally, rely on onChange-based validation (triggered after touched flag is set). The first blur will be "missed" but validation runs on the next change.
    - Choose option (b): validate on change when touched. Remove handleBlur calls since FloatingInput does not expose onBlur. The validation UX is nearly identical -- errors appear as the user types after first interaction.
    - NOTE: If this creates issues, an alternative is to pass a wrapper div with onBlur around FloatingInput. But for this phase, onChange-based validation after touched is sufficient.
