  <name>Task 1: Restructure TransactionForm with bottom sheet layout</name>
  <files>src/components/transactions/TransactionForm.tsx</files>
  <action>
    Major restructure of TransactionForm.tsx. The form transforms from a standard form layout inside Modal into an immersive bottom sheet with distinct visual zones. Keep the same server action integration (createTransaction/updateTransaction) and validation logic.

    **Outer wrapper (TransactionForm component):**
    - Still wraps content in `<Modal>`, but now uses `headerContent` prop instead of `title` prop.
    - The headerContent renders a custom header row:
      ```tsx
      <div className="flex items-center justify-between px-7 pb-2">
        <button type="button" onClick={onClose} className="rounded-full p-2 text-text-tertiary hover:text-text-primary transition-colors duration-200" aria-label="Cerrar">
          <X size={20} />
        </button>
        <span className="text-sm font-medium text-text-primary">
          {transaction ? 'Editar movimiento' : 'Nueva transaccion'}
        </span>
        <button type="button" onClick={handleSave} disabled={submitting} className="text-xs font-semibold uppercase tracking-[2px] text-accent hover:text-accent-hover transition-colors duration-200 disabled:opacity-50">
          {showCheckmark ? <Check size={20} className="text-accent" /> : 'GUARDAR'}
        </button>
      </div>
      ```
    - NOTE: The headerContent needs access to handleSave and submitting state, so it must be rendered inside TransactionFormContent (which has the state). Move the Modal wrapper INTO TransactionFormContent, or lift the headerContent into TransactionFormContent's render. The cleanest approach: TransactionFormContent renders `<Modal>` itself with headerContent that references its own state.
    - Restructure: TransactionForm (outer) becomes a thin wrapper that just conditionally renders TransactionFormContent with the key. TransactionFormContent renders the Modal.

    **Section 1: Hero Amount Zone**
    - A div with `dot-matrix-hero` class (CSS pseudo-element overlay from globals.css) and `bg-surface-elevated rounded-2xl p-6 mb-4`.
    - Inside (with `relative z-[2]` to render above the dot-matrix overlay):
      - Large monospaced amount display: `<span className="font-mono text-4xl font-bold text-text-primary tabular-nums">${formatAmountDisplay(amount) || '0.00'}</span>`
      - The "$" prefix should use the MoneyAmount muted style: wrap in a span with smaller size and text-text-tertiary. Example: `<span className="font-mono text-2xl text-text-tertiary mr-1">$</span><span className="font-mono text-4xl font-bold text-text-primary tabular-nums">{formatAmountDisplay(amount) || '0.00'}</span>`
      - Center the amount display: `text-center`
    - The amount is NOT an `<input>`. It is a display-only span controlled by the Numpad below.

    **Section 2: Type Toggle (TogglePills)**
    - Replace the manual two-button toggle with TogglePills component:
      ```tsx
      <TogglePills
        options={[
          { value: 'EXPENSE', label: 'Gasto' },
          { value: 'INCOME', label: 'Ingreso' },
        ]}
        value={type}
        onChange={handleTypeChange}
        className="justify-center mb-4"
      />
      ```
    - `handleTypeChange` stays the same (resets categoryId when type changes).

    **Section 3: Category Grid**
    - Replace the current 3-column grid with a 4-column circular icon grid:
      ```tsx
      <div className="grid grid-cols-4 gap-3 mb-4">
        {filteredCategories.map((cat) => (
          <button key={cat.id} type="button" onClick={() => setCategoryId(cat.id)}
            className="flex flex-col items-center gap-1.5"
            aria-label={cat.name}
          >
            <div className={cn(
              'flex h-12 w-12 items-center justify-center rounded-full transition-all duration-200',
              categoryId === cat.id
                ? 'ring-2 ring-accent'
                : ''
            )}
              style={{ backgroundColor: `${cat.color}26` }}
            >
              <DynamicIcon name={cat.icon} size={24} style={{ color: cat.color }} />
            </div>
            <span className="text-[11px] text-center text-text-secondary truncate w-full">
              {cat.name}
            </span>
          </button>
        ))}
      </div>
      ```
    - Key change: icons are in 12x12 (h-12 w-12) circles, selection state is a 2px chartreuse ring (`ring-2 ring-accent`), not background fill. No border on the button.
    - Category error message stays below the grid.

    **Section 4: Numpad**
    - Render `<Numpad value={amount} onChange={setAmount} />` imported from `./Numpad`.
    - The Numpad handles all amount string manipulation.
    - Remove the old amount `<input>` entirely. No `<input type="text" inputMode="decimal">` for amount.
    - Remove `displayAmount` state and `formatAmountDisplay`/`cleanAmountInput` for the amount field. The amount is just a raw string managed by Numpad.
    - Keep `formatAmountDisplay` as a local function for the hero display (converts "15075" style number to comma-formatted display -- but actually, Numpad works with peso values like "150.75", not centavos). The hero display shows the raw amount string with commas. If amount is "1500", display "1,500". If "150.75", display "150.75".
    - Use the existing `formatAmountDisplay` function for the hero zone display.
    - Add `className="mb-4"` wrapper around Numpad.

    **Section 5: Optional Fields (collapsible)**
    - Keep the "Mas detalles" collapsible toggle.
    - Inside, replace all `<input>` elements with `<FloatingInput>`:
      - Description: `<FloatingInput label="Descripcion" value={description} onChange={setDescription} optional />`
      - Date: `<FloatingInput type="date" label="Fecha" value={date} onChange={setDate} />`
      - Payment method (expense only): Keep as pill button grid (NOT an input, so stays as-is).
      - Notes: For the textarea, FloatingInput does not support textarea. Keep as a styled `<textarea>` with underline-only styling: `className="w-full bg-transparent border-0 border-b border-border-divider py-2 text-sm text-text-primary focus:border-accent outline-none transition-colors duration-200 resize-none"` with a label above.
      - Income source (income only): Keep as `<select>` since FloatingInput does not handle select dropdowns. Style the select with underline-only: `className="w-full bg-transparent border-0 border-b border-border-divider py-2 text-sm text-text-primary focus:border-accent outline-none transition-colors duration-200"`.

    **Save behavior:**
    - Add `showCheckmark` boolean state, initially false.
    - On successful submit: set `showCheckmark = true`, wait 200ms, then call `onClose()`, then toast.
    - ```tsx
      if ('success' in result) {
        setShowCheckmark(true)
        setTimeout(() => {
          onClose()
          toast.success(isEditing ? 'Movimiento actualizado' : 'Movimiento registrado')
        }, 200)
      }
      ```
    - The GUARDAR button in headerContent shows a `<Check>` icon (from lucide-react) when `showCheckmark` is true.

    **Remove the bottom submit button:**
    - The old full-width "Guardar" button at the bottom is replaced by the "GUARDAR" text in the header. Remove the `<button type="submit">` at the bottom.
    - Since the form no longer uses `<form onSubmit>`, change the form element approach:
      - The entire content can remain wrapped in a `<form>` for semantic purposes, but submission is triggered by the GUARDAR button's onClick (calling handleSave directly), not by form submission event.
      - OR: keep `<form onSubmit={handleSubmit}>` and make GUARDAR a `type="submit"` button. This is simpler and preserves Enter key submission.
      - Choose: keep `<form onSubmit={handleSubmit}>`, make GUARDAR `type="submit"`. But GUARDAR is in headerContent (outside the form element in DOM). This won't work.
      - Solution: Make GUARDAR call handleSave directly via onClick. Remove `<form onSubmit>` wrapper. The content sections are just divs inside Modal. handleSave contains the same logic as the current handleSubmit but without `event.preventDefault()`.

    **Validation:**
    - Amount validation: validate on save. If amount is empty or "0", show error.
    - Category validation: if no category selected, show error on the category section.
    - Keep the Zod schema validation on submit (createTransactionSchema.safeParse).
    - Convert amount from peso string to centavos string using toCents: `toCents(amount || '0')`.

    **Edit mode:**
    - When editing (transaction prop provided), pre-fill:
      - amount: convert cents to pesos string (Number(transaction.amount) / 100).toString()
      - type, categoryId, description, date, paymentMethod, notes, incomeSourceId: same as current
    - Title shows "Editar movimiento" in the header.
    - The Numpad shows the pre-filled amount and allows editing.
