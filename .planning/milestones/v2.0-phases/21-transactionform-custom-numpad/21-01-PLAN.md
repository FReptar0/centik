  <behavior>
    - Renders a 4-column CSS grid with 13 keys (1-9, 0, decimal, 00, backspace, plus empty cells)
    - Layout rows: [1, 2, 3, backspace], [4, 5, 6, decimal(.)], [7, 8, 9, 00], [empty, 0, empty, empty]
    - All digit buttons use IBM Plex Mono (font-mono class)
    - Backspace key shows Lucide Delete icon instead of text
    - Each key has minimum 48px touch target (min-h-[48px] min-w-[48px])
    - Keys styled: bg-surface-elevated, text-text-primary, rounded-xl, transition-colors
    - Key press state: bg-surface-hover with 100ms transition
    - Empty cells are invisible div placeholders (no button)
    - All buttons have type="button" (prevent form submission)
    - Component is "use client" (has click handlers)

    Amount string manipulation logic:
    - appendDigit(d): Appends digit to value. No-op if already 2 decimal places after "."
    - appendDecimal(): Appends "." only if value does not already contain "."
    - appendDoubleZero(): Appends "00" to value. If value is empty or "0", stays "0". Respects max 2 decimal places.
    - backspace(): Removes last character. If value becomes empty, stays empty (parent controls display of "$0.00").
    - Leading zeros: "00" on empty produces "0" (not "00"). But "1" + "00" = "100".
    - Decimal edge: "." alone is valid (user starting with decimal). "0." is valid.
    - Max check: if value contains "." and already has 2 chars after ".", digit append is no-op.

    Test cases (RED phase -- write all tests FIRST):
    1. Renders 10 digit buttons (0-9) plus decimal, 00, backspace = 13 buttons total
    2. All digit buttons have type="button"
    3. Tapping 1, 2, 3 in sequence produces onChange calls with "1", "12", "123"
    4. Tapping decimal when no decimal exists appends "."
    5. Tapping decimal when decimal already exists is a no-op
    6. Tapping backspace removes last character
    7. Backspace on empty string stays empty
    8. Tapping 00 appends two zeros: "1" -> "100"
    9. Tapping 00 on empty value stays "0" (no leading zeros)
    10. Max 2 decimal places: "1.23" + "4" = no change (still "1.23")
    11. Decimal then digits: "" -> "." -> ".5" works
    12. Double-zero after decimal respects limit: "1." + "00" = "1.00", "1.5" + "00" = "1.50" (only 1 zero added)
    13. Backspace icon renders for backspace key (accessible via aria-label)
    14. All buttons have minimum 48px height (min-h class)
  </behavior>
